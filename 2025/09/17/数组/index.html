算法详解：二分查找的原理与实现

lc704：二分查找

题目链接：力扣T704

题解

思路

在这个题的情境下，显然给出的区间是一个左闭右闭的区间，那么left <= right就是有效的，因为这样一定能确保排查的全面，即使left和right重合之后，也可以查那个元素是否为target

然后还有一个关键的地方，就是每一轮你查完之后，如果mid下标的值不是target的话，可以直接扔掉现在的这个mid元素，所以处理新的left/right的时候才有一个+1/-1的情况

相关题目

1.力扣35：搜索插入元素
这个题跟前面的题唯一的区别就是多了一个最后应该插入的位置，其实这里的结论是插入位置的下标就是left，下面我们详细的解释这个事

定义：令 pos 为应插入的位置（数组中第一个 >= target 的下标；若没有则 pos = n）。

不变式：每次循环开始时，总有
left ≤ pos ≤ right + 1。
（初始 left=0, right=n-1，显然 pos∈[0,n]，成立。）

维护（取 mid = ⌊(left+right)/2⌋）：

若 nums[mid] < target，说明插入点在 mid+1 之后，于是把 left = mid+1，不变式仍成立（新区间包含 pos）。

若 nums[mid] ≥ target，说明插入点不在 mid+1..，插入点 ≤ mid，于是把 right = mid-1，不变式仍成立。

结束：循环在 left > right 时退出。结合不变式得 pos ∈ [left, right+1] 且 left > right，因此只能是 pos = left。所以返回 left 就是正确的插入位置。

result：最终的left对应了搜索元素要插入的位置

数组元素删除

题目简述：给定一个数组nums和一个特定值target，删除这个数组当中所有值为target的元素然后返回原来的数组

题解

双指针法：定义两个指针，分为快指针和慢指针。其中快指针用来遍历数组所有元素，慢指针用来记录所有需要删除的元素，同时慢指针的下表代表了数组中其他元素正确的排列位置，同时最后慢指针的idx + 1就是新数组的长度

有序数组的平方

题目简述：给定一个按非递减顺序排序的整数数组nums，返回每个数字的平方组成的新数组，并按非递减顺序排序

题解

双指针法的另一个应用，但这里指针就没有快慢之分了，两个指针分别指向数组的两端，然后比较二者的平方值，较大的那个就放到新数组的最后面，然后对应的指针向中间移动一位，直到两个指针相遇

长度最小的子数组

题目简述：给定一个含有n个正整数的数组和一个正整数target，找出该数组中满足其和≥target的长度最小的连续子数组，并返回其长度。如果不存在符合条件的子数组，返回0

题解

双指针法的变形：这里的双指针其实是一个滑动窗口的概念，定义两个指针left和right，初始都指向数组的第一个元素，然后right指针不断向右移动，直到窗口内的元素和大于等于target，然后记录下当前窗口的长度，然后left指针向右移动一位，继续判断窗口内的元素和是否大于等于target，如果是则继续更新最小长度，否则继续移动right指针，直到right指针到达数组末尾

区间和（前缀和）

题目描述：给定一个整数数组 Array，请计算该数组在每个指定区间内元素的总和。

题解：使用前缀和优化时间复杂度，然后求和的部分就可以使用前缀和来代替，注意下标是否正确
